Projects

Genesis GV80

TFT lcd panel with touch screen and haptic function which is firstly adapted in HKMC automotive

    What I do
  • Programmed touch IC driver(SPI, I2C), application, diagnosis, update function on RTOS system. And integrated these modules to platform
  • Geusture algorithm development - Wheel, Button, Drag, Flicker
  • Implemented coordinates compensation algorithm by using rotation matrix for misaligned sensor attachment
  • Designed module data structure, flow chart, dynamic view
  • Automated code generation of touch coordinates by using VBA programming
  • Documentate touch specification and verify touch function
Challenges and Solutions

  • Challenge1 : Typing touch coordinates manually according to a number of variant is too time consuming works and it was difficult to find error.
    Solution1 : Implemented auto code generation and verification tool by using VBA programming. Working time is saved from 1day to 1hr.

  • Challenge2 : There was no specification for touch function. So, we had always arguments with customer and cooperator.
    Solution2 : Studied materials of more than 1,000 pages and documentated specification according to touch IC internal function in detail. After establishing test procedure, verified and tuned complete product with cooperator based on this document by using robot arm machine. We satisfied customer requirments.

IONIQ HEV/EV

FATC controller with button and slider touch function

    What I do
  • Programmed driver(SPI,NVM,ADC,POR) and app software of 32-bit, two 8-bit microprocessor at the same time
  • Developed algorithm of extracting position coordinates from just 3-node rotary sensors and changeable threshold algorithm by using interpolation
  • Realized FBL in 8bit micom, Gateway function in 32bit micom, external update tool software based on USD on CAN
  • Implemented rotary geusture algorithm to control temperature set
  • Made filter algorithm and partial burst function to improve noise immunity of touch analog signal
  • Did Script programming which merge FBL and app with validation checking according to memory structure
Challenges and Solutions

  • Challenge1 : Because rotary touch sensors are samll. So, sensitivity was very poor and not uniform according to position to provide reliability function.
    Solution1 : Checked sensitivity from edge to edge at interval of 1mm. And then made the array table according to possition. As developing algorighm by using interpolation, sensitivity could be uniformed and improved.

  • Challenge2 : Becasue used severial microprocessor MIPS are different, communication timming was problem, when commucating between each microprocessor and updating complete product.
    Solution2 : To improve timming issue, I designed update system and set one IO port which infrom processing(store, delete, load) of slow microprocessor. Main microprocessor check this port before starting communication. And also, checked max latancy time in full overload.

The K9

FATC panel with touch proximity and interactive illumination function. This function is firstly adapted in HKMC automotive

    What I do
  • Programmed driver(I2C,ADC) and app software of 16-bit, 8-bit microprocessor at the same time
  • Implemented software dimming control which detects analog signal and convert to pwm pulse with desired freq and duty
  • Designed and developed algorithm such as filter(1D Kalman, Butterworth, Log), analog signal boosting(Pre-emphasis), Tracking which follow original signal(IIR)
  • Realized FBL in 8bit micom, Gateway function in 16bit micom, external update tool software based on USD on CAN
  • Did Script programming which merge FBL and app with validation checking according to memory structure
Challenges and Solutions

  • Challenge1 : Because proximity touch sensor was very small and hardware pcb didn't provide reliability in the situation of current flow fluctuation, sensitivity was very poor and there are a lot of noise which caused false function.
    Solution1 : To improve sensitivity, I increased software gain. And in order to suppress noise which casued by increased gain, I implemented filter combination algorithm(Median + Kalman). After that, to speed up response time of signal, I developed pre-emphasis algorithm which have different weight according to specified range.

  • Challenge2 : When I communicated with 8bit microprocessor by using I2C, there was stop phenomenon of one's processing in initial stage, for this one has only 1 interrupt vector and I2C interrupt signal disturb mian touch ADC processing.
    Solution2 : Checked free time between end of ADC processing and restart of one. This time was indicated by IO port which connected between two processors. I got I2C communication of 1 byte to perform only during this time.
